Функция редактирования заказа позволит пользователям изменять уже созданные заказы в приложении "Частная пекарня". Это может включать добавление, удаление или изменение количества продуктов в заказе, а также обновление информации о доставке и способе оплаты.


Цели функции:

1. Обеспечить пользователям возможность управлять своими заказами.

2. Упростить процесс внесения изменений в заказы.

3. Поддерживать актуальность информации о заказах в системе.


Экран списка заказов:

Пользователь должен видеть список своих текущих заказов с возможностью выбора конкретного заказа для редактирования.

  
Экран редактирования заказа:

  Отображение деталей заказа, включая:

    1. Список заказанных продуктов с их названиями, изображениями и ценами.

    2. Возможность изменить количество каждого продукта (с использованием ползунков или полей ввода).

    3. Кнопка для удаления продукта из заказа.

    4. Поле для добавления новых продуктов (с возможностью поиска по каталогу).

    5. Обновление информации о доставке (адрес, время доставки).

    6. Выбор способа оплаты.

  Добавление продукта:

    Пользователь может выбрать продукт из каталога и добавить его в заказ.

  Изменение количества продукта:

    1. Пользователь может изменить количество уже добавленных продуктов. При изменении количества должно автоматически пересчитываться общее значение заказа.

  Удаление продукта:

    2. Пользователь может удалить продукт из заказа. После удаления продукт должен исчезнуть из списка, и общая сумма заказа должна обновиться.

  Обновление информации о доставке:

    Пользователь может изменить адрес доставки и время, если это необходимо.

  Сохранение изменений:

    После внесения всех изменений пользователь должен иметь возможность сохранить их, нажав кнопку "Сохранить изменения".

  
Отмена редактирования:

  Пользователь должен иметь возможность отменить редактирование и вернуться на предыдущий экран без сохранения изменений.


Валидация данных:

  1. Проверка на корректность введенной информации (например, адреса доставки).

  2. Проверка наличия выбранных продуктов на складе перед добавлением их в заказ.

  3. Уведомление пользователя о любых ошибках или конфликтах (например, если количество запрашиваемого продукта превышает доступное).


Нефункциональные требования:

  1. Производительность: Функция редактирования должна работать быстро и не вызывать задержек при обновлении данных.

  2. Безопасность: Все изменения должны быть защищены от несанкционированного доступа. Необходимо использовать аутентификацию пользователя.

  3. Удобство использования: Интерфейс должен быть интуитивно понятным и простым в навигации.


Технические требования:

  1. API: Разработать RESTful API для обработки запросов на редактирование заказа (GET, POST, PUT, DELETE).

  2. База данных: Обновление соответствующих таблиц в базе данных (например, Order, OrderedProduct) при изменении заказа.

  3. Клиентская часть: Реализация интерфейса редактирования на мобильном приложении с использованием подходящих технологий (например, React Native, Flutter).


Тестирование:

  1. Провести тестирование функции редактирования заказа, включая юнит-тесты и интеграционные тесты для проверки всех сценариев использования.

2. Проверить корректность работы функции на различных устройствах и версиях операционных систем.
